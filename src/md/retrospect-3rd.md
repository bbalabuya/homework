# 자신감 획득

나는 VS Code에서 **'Better Comments'**라는 확장 프로그램을 사용한다. 주석마다 색깔을 지정할 수 있어서 중요한 포인트를 눈에 띄게 관리할 수 있고, 코딩할 때 훨씬 편하다는 장점이 있다.

하지만 한 가지 문제가 있었다. 기본 주석의 색이 너무 어두운 회색이라 눈에 잘 들어오지 않았던 것. 색을 밝게 바꾸고 싶어서 Better Comments 공식의 설정방법도 찾아보고 코드도 건드려봤지만, 바뀌질 않는다.
![](https://velog.velcdn.com/images/why_does_it_work/post/7c8f2d6e-122b-4ad0-8c1b-050703f4ca7e/image.png)

결국 제미나이에게 Better Comments 설정 파일을 보여주니 바로 원인을 파악하고 수정 코드를 주었다. 하지만 코드를 적용해도 여전히 색이 변하지 않아 포기하려 했다. AI가 새로운 대안을 제시했다. 확장 프로그램 설정을 건드리지 말고, 아예 VS Code 자체 설정에서 주석 색상을 강제로 지정하라는 것이었다.

우선 사용 중인 테마 이름을 알려주니 다음과 같은 코드를 주었다.

```
 "editor.tokenColorCustomizations": {
    "[One Monokai]": {
      "comments": "#FFFFFF", // 일반 주석을 흰색으로
      "variables": "#00FF00"  // 변수 이름을 초록색으로 (예시)
    }
 }
```

settings.json 파일 맨 밑에 이 코드를 추가했더니, 짜잔!
![](https://velog.velcdn.com/images/why_does_it_work/post/e33bff8b-122d-473f-a8fc-e47eb76d8b4e/image.png)

색이 바로 바뀌는 마법 같은 일이 일어났다. VSCode에서 확장 프로그램을 설치하는 것 외에 설정을 직접 건드린 건 이번이 처음인 것 같다. 개발의 세계에서는 오타 하나, 실수 하나만으로도 바로 오류가 나다 보니 그동안 무언가를 직접 바꾸기가 겁나서 시도조차 하지 않았었다.

그런데 이번엔 뭐랄까... 겁이 없어졌다고 해야 할까, 자신감이 붙었다고 해야 할까? 이제는 내가 무엇을 수정하는지 알아서인지 괜찮은거 같다. 아무튼, 이제 설정 파일 건드리는게 무섭지 않을거 같다.

# 무한로딩

> 주어진 실습시간 15분. 멍때리기 10분.

엄.. 내가 고장난 기분이다. 강사님께서 주신 시각 자료를 배운걸 토대로 만들기만 하면 된다.
![](https://media3.giphy.com/media/v1.Y2lkPTc5MGI3NjExYmMxYXl3OXNkMTdpa3ZmdDJwNGxrOTBvcWQxMjdlZTJrZzZkcDM3dCZlcD12MV9pbnRlcm5hbF9naWZfYnlfaWQmY3Q9Zw/8tASnpboXPdPSm4PAK/giphy.gif)

그런데 그게 쉽지가 않다. 정확히 말하자면 순서를 모르겠다. 배웠던 것들 중에서 어떤 기능을 이용하여 어떤 순서로 만들어 나가야 할지 설계하는게 쉽지 않다. 어떤 태그를 이용하고 어떤 속성을 써야하는지 시각자료를 보면 머리에 아이디어가 쌓이지 않고 반대로 비워진다.

시각 자료를 본 첫 3초 동안은 display 속성 주고 padding 주고 font-size 주고 딱 거기까지😭
하다보면 늘겠지만 아직은 어렵다. 시간 압박이 있어서 그러기도 할거고 아직은 배운 것만 이용해서 써야하다 보니 지금은 머리가 안 굴러군다.

앞으로 나는 시각자료가 주어지면 일단 타이핑 해보는 습관을 들일 필요가 있어보인다. 멍 때리면서 엄.. 이게.. 뭐더라.. 하는 것보단 일단 아는거라도 꺼내 쓰는게 중요해보인다.
