# 내 실력에 기반 다지기

이번 주는 모든 환경 설정을 끝내고 진도를 아주 빠르게 나갔다. HTML만 한 달이 걸릴 것 같았으나 눈 떠보니 일주일이 지났고, 벌써 CSS 파트를 나가고 있다. 여기서 제대로 느낀 점은

> 아! 나란 놈은 프론트엔드 개발자면서 정말 대충 하고 있었군.

HTML은 태그로 뼈대 구조 잡고, CSS로 있어 보이게 한 다음, JS로 기능 만들면 끝인 줄 알았다. 하지만 그게 아니었다.

## 좋은 개발자란 누구인가

- 버그 없이 잘 굴러가도록 하는 개발자?
- 말하는 대로 화려하게 만들어오는 개발자?
- 뚝딱뚝딱 5분 만에 만들어 오는 개발자?

다 아닌 것 같다. 지원하지 않는 구형 브라우저까지 고려하고, 신체적 제약에 상관없이 누구나 웹에 접속해 정보를 평등하게 볼 수 있게 하는 개발자. 이런 개발자야말로 참된 개발자인 것 같다. 코딩 기초가 부족해서 부트캠프를 수강하려고 한 건데, 코딩 기초는커녕 '개발자로서의 철학' 자체가 부족했음을 느낀다.

# 프론트엔드 개발자 누가 쉽다고 했어!

![내 심정](https://velog.velcdn.com/images/why_does_it_work/post/06cb0217-266f-4cb6-87df-af28edd1da8e/image.jpg)

진짜 누가 쉽다고 했어🤯
솔직히 대학교 1, 2학년 때는 개발에 대해 아는 게 없었다. 주변에서 주워듣기로 백엔드 개발자는 서버 설계부터 구축, 보안, 배포까지 다 신경 써야 한다길래 “아, 나 같은 놈이 백엔드 가면 망하겠구나” 싶어서 프론트엔드로 진로를 잡았다.
하지만 이번에 공부하며 느낀 점은, 웹 페이지가 단순히 작동하는 것과는 별개로 프론트엔드가 얼마나 섬세하고 완벽해야 하는가에 대한 것이었다. HTML을 만만하게 본 적은 없지만, 생각 이상으로 중요하다는 것을, 그리고 내가 신경 써야 할 영역이 얼마나 넓은지를 절실히 느낀다.

# 마무리

이번 한 주 동안 HTML의 구조와 '누구나 접근할 수 있게 설계하는 것'이 얼마나 중요한지를 아주 빠르게 배웠다. 프론트엔드도 결코 쉬운 길이 아니며, 오히려 사람에 대한 깊은 이해가 필요한 분야라는 것을 깨닫는 값진 시간이었다.
